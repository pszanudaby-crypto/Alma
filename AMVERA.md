# Деплой на Amvera

В репозитории есть `Dockerfile` и `amvera.yml`. Amvera соберёт контейнер, запустит `npm start` и примонтирует постоянное хранилище в `/data`.

## Переменные и секреты

В Amvera добавьте:

- `DATABASE_URL` — строка подключения к PostgreSQL.
- `JWT_SECRET` — длинная случайная строка.
- `ADMIN_EMAILS` — email админов через запятую.
- `CORS_ORIGIN` — домен фронтенда, если фронтенд останется на Vercel.
- `PUBLIC_APP_URL` — публичный URL приложения/API на Amvera.
- `MEDIA_DIR=/data/uploads`

## Первый запуск

После создания PostgreSQL выполните миграции:

```bash
npm run migrate
```

Если Amvera запускает только основное приложение, миграцию можно выполнить локально с тем же `DATABASE_URL`, либо через отдельный временный запуск команды в панели/терминале Amvera.

Первый зарегистрированный пользователь автоматически получает роль `admin`.
